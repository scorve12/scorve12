## 💡 핵심 학습 및 인사이트

### 1. **도메인 전문가와의 소통**

```
문제 상황:
대표님: "당연히 법정동 주소를 써야죠!"
개발팀: "법정동이 뭐죠...? 🤔"

해결 과정:
1. 용어 사전 작성 (50+ 농업/드론 용어)
2. 프로토타입 기반 소통 (말보다 화면)
3. 플로우차트로 복잡한 로직 시각화
4. "왜 이렇게 해야 하는지" 비즈니스 맥락 이해

💡 학습 포인트:
- 도메인 지식 없이는 좋은 설계 불가능
- 기술 전문가 ≠ 도메인 전문가
- 질문을 많이 하는 것이 더 빠른 길
```

### 2. **작은 팀의 높은 생산성**

```
4명으로 6개월 만에 프로덕션 배포 비결:

🤖 자동화
- CI/CD 파이프라인 (GitHub Actions + AWS)
- API 테스트 자동화 (Postman + Newman)
- 배포 자동화 (Elastic Beanstalk)
→ 1일 1시간 절약 = 월 20시간 절약

📚 문서화
- Notion 적극 활용 (회의록, API 문서, 용어 사전)
- Swagger 자동 생성 (DRF Spectacular)
- README, 주석, PR 템플릿
→ 소통 비용 30% 절감

🛠 도구 활용
- GitHub Copilot (코드 작성 속도 2배)
- ChatGPT (문서 작성, 디버깅)
- Figma to Notion (디자인 스펙 자동 전달)
→ 반복 작업 시간 50% 절약

💡 학습 포인트:
"70% 완성도로 빠르게 출시 후 반복 개선"
```

### 3. **인증 방식 전환 결정**

```
🚨 위기 상황 (개발 3개월 차):
✅ 세션 기반 인증 이미 완성
❌ APP 담당자: "모바일에서 세션 관리 너무 어려워요"
⏰ 남은 기간: 3개월

의사결정 프로세스:

1. 현재 상황 분석
  - 백엔드: 1주일 소요
  - 프론트: 2주 단축 가능
2. 리스크 평가
  - 전환 실패 시 일정 지연
  - 전환 안 하면 품질 저하
3. 팀 논의 
  - 모두가 JWT 전환 동의
  - 1주일 집중 개발 결정
4. 결과
  ✅ 1주일 만에 테스트 및 검증 완료
  ✅ 기존의 WEB 및 APP 코드 재사용성 상승

💡 학습 포인트:
- 이미 만든 코드를 버리는 용기
- Sunk Cost Fallacy 극복
- 장기적 생산성 vs 단기적 노력
```
